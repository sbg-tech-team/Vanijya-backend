## 1. Module Overview

The groups module handles:

- **Group creation** — only verified users can create groups. Creator is automatically set as admin.
- **Group management** — update group info, permissions, icon. Admin-only.
- **Membership** — join public groups directly, join private groups via invite link, leave, freeze/unfreeze members (admin), mute/favorite (per-user preference).
- **Invite links** — admin generates a unique token; anyone with the token can join.
- **Recommendations** — personalised group suggestions based on the user's profile using cosine + activity reranking.

---

## 2. How User Identity Works

All groups endpoints require `Authorization: Bearer <token>`. The acting user's identity is derived from the JWT — **never** from a `?user_id=` query parameter.

```
POST /api/v1/groups/
Authorization: Bearer <access_token>

GET  /api/v1/groups/{group_id}
Authorization: Bearer <access_token>
```

> **Migration note (2026-05-12):** The old `?user_id=<uuid>` query parameter has been removed from every endpoint. The old `GET /api/v1/groups/suggestions/{user_id}` path parameter has also been removed — use `GET /api/v1/groups/suggestions` with a Bearer token instead.

---

## 3. Verification Gate — Who Can Create Groups

`POST /api/v1/groups/` checks that the user's profile has `is_verified: true`. If not:

```json
{ "detail": "Only verified users can create groups. Complete profile verification first." }
```

**Status:** `403 Forbidden`

All other group endpoints are open to any user.

| Column | Type | Notes |
|---|---|---|
| `group_id` | UUID PK FK → groups.id | |
| `messages_24h` | INTEGER | Messages in last 24 hours |
| `unique_senders_24h` | INTEGER | Distinct senders in 24h |
| `active_members_7d` | INTEGER | Members active in last 7 days |
| `member_growth_7d` | INTEGER | Net new members in 7 days |
| `updated_at` | TIMESTAMP | |

### `group_embeddings` — 11-dim IS vector

| Column | Type | Notes |
|---|---|---|
| `group_id` | UUID PK FK → groups.id | |
| `embedding` | **VECTOR(11)** | 11-dim pgvector column — replaces JSONB as of migration `a3b4c5d6e7f8` |
| `updated_at` | TIMESTAMP | |

**HNSW index** (cosine distance, created by migration `a3b4c5d6e7f8`):
```sql
CREATE INDEX ix_group_embeddings_hnsw
  ON group_embeddings USING hnsw (embedding vector_cosine_ops);
```
This index powers the ANN search in the recommendation engine — without it the engine falls back to a full table scan.

```

Content-Type: application/json
```

**Request body:**
```json

    "name": "Sugar Traders Maharashtra",
    "description": "A group for sugar traders across Maharashtra to discuss market rates.",
    "group_rules": "No spam. Only trade-related discussions.",
    "commodities": ["sugar", "rice"],

    "accessibility": "public",
    "posting_perm": "all_members",
    "chat_perm": "all_members",

```
POST https://vanijyaa-backend.onrender.com/api/v1/groups/?user_id=c37a3257-dc3f-43be-9fb0-33cf918b11ff

## 10. Group Recommendations

### `GET /api/v1/groups/suggestions/{user_id}`

Returns top 20 group suggestions ranked by semantic similarity (75%) + activity score (25%). Excludes groups the user is already in and `private` groups.

**Two-stage pipeline:**

```
Stage 1 — HNSW ANN (SQL, fast)
  pgvector <=> operator on group_embeddings
  Fetches top 80 semantically closest groups, already filtered:
    - accessibility != 'private'
    - group not already joined by user
  Cost: O(log N) — independent of total group count

Stage 2 — Activity reranking (Python, cheap)
  For each of the 80 candidates, blends:
    semantic_score × 0.75  +  activity_score × 0.25
  activity_score = log1p(messages_24h × 0.4
                       + active_members_7d × 0.4
                       + member_growth_7d × 0.2) / log1p(100)
  Returns top 20 after sorting
```

**Why two stages?** HNSW gives you the most relevant groups by profile match in microseconds. Activity reranking then surfaces groups that are also *active*, not just semantically close.

**Example:**
```
GET https://vanijyaa-backend.onrender.com/api/v1/groups/suggestions/c37a3257-dc3f-43be-9fb0-33cf918b11ff
```

**Success `200`:**
```json
{
    "success": true,
    "message": "Group suggestions fetched",
    "data": [
        {
            "group": { "id": "...", "name": "Sugar Traders Maharashtra", "member_count": 14 },
            "match_score": 0.8004,
            "match_reasons": ["Matches your commodities", "Targets your role"]
        }
    ]
}
```

## 12. Enum Values Reference

| Field | Allowed values |
|---|---|
| `accessibility` | `public`, `private`, `invite_only` |
| `posting_perm` | `all_members`, `admins_only` |
| `chat_perm` | `all_members`, `admins_only` |
| `category` | `commodity_trading`, `news`, `network` |
| `commodities` items | `"rice"`, `"cotton"`, `"sugar"` |
| `target_roles` items | `"trader"`, `"broker"`, `"exporter"` |
| `member_role` | `admin`, `member` |

**Accessibility rules:**
- `public` — anyone can join via `POST /{group_id}/join?user_id=`
- `private` — visible in list but join only if added by admin
- `invite_only` — join only via invite link token
